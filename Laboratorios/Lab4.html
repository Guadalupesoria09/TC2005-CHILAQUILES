¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Diferencias: Java es un lenguaje de programación orientado a objetos, independiente de la plataforma, y se utiliza para crear aplicaciones de escritorio, móviles y web. JavaScript es un lenguaje de programación que se ejecuta principalmente en el navegador web, utilizado para crear interactividad en las páginas web. Mientras que Java es compilado, JavaScript es interpretado.

Semejanzas: Ambos lenguajes comparten una sintaxis similar y se utilizan en desarrollo web, aunque en diferentes contextos.

¿Qué métodos tiene el objeto Date? (Menciona al menos 5)

  • getFullYear(): Devuelve el año de la fecha especificada según la hora local.
  • getMonth(): Devuelve el mes de la fecha especificada según la hora local (de 0 a 11).
  • getDate(): Devuelve el día del mes de la fecha especificada según la hora local.
  • getDay(): Devuelve el día de la semana de la fecha especificada según la hora local (0 es domingo, 6 es sábado).
  • getHours(): Devuelve la hora de la fecha especificada según la hora local.

¿Qué métodos tienen los arreglos? (Menciona al menos 5)

  • push(): Añade uno o más elementos al final de un arreglo y devuelve la nueva longitud del arreglo.
  • pop(): Elimina el último elemento de un arreglo y lo devuelve.
  • shift(): Elimina el primer elemento de un arreglo y lo devuelve, desplazando todos los elementos restantes.
  • unshift(): Añade uno o más elementos al inicio de un arreglo y devuelve la nueva longitud del arreglo.
  • slice(): Devuelve una copia superficial de una porción de un arreglo en un nuevo arreglo, comenzando por el inicio hasta el fin (sin incluir el fin).

¿Cómo se declara una variable con alcance local dentro de una función?

Una variable con alcance local dentro de una función se declara utilizando las palabras clave var, let o const dentro de la función. Ejemplo:

function miFuncion() {
     let variableLocal = "Hola Mundo";
     console.log(variableLocal); // "Hola Mundo"
 }

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Utilizar variables globales dentro de funciones puede llevar a errores difíciles de detectar, ya que cualquier función puede modificar accidentalmente el valor de una variable global, lo que podría alterar el comportamiento del programa. Además, el uso excesivo de variables globales puede llevar a una mayor dificultad en la depuración y mantenimiento del código, así como a problemas de rendimiento, ya que las variables globales permanecen en memoria durante la vida del programa.
